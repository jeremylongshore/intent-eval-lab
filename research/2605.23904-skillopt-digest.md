## Core thesis (verbatim from abstract)

> "SkillOpt is, to our knowledge, the first systematic controllable text-space optimizer for agent skills"

Skills are treated as frozen agent external state. A separate optimizer model converts scored rollouts into text edits. Zero inference-time model calls at deployment.

## Mechanism (verbatim)

- "Bounded add/delete/replace edits on a single skill document"
- Acceptance gate: "strictly improved held-out validation scores"
- "Zero inference-time model calls at deployment"
- Stability mechanisms: "textual learning-rate budget, rejected-edit buffer, and epoch-wise slow/meta update"

## Benchmarks reported

- 6 benchmarks
- 7 target models
- 3 execution harnesses: direct chat, Codex, Claude Code
- **+23.5pp on GPT-5.5 in direct chat**
- **+24.8pp in Codex**
