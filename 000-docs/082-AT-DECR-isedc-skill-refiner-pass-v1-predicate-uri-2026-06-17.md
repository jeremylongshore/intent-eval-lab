## 1. Mission of this record

Mint the `skill-refiner-pass/v1` predicate URI — the in-toto attestation the Skill Refiner emits when a real SkillVersion clears the `@j-rig/refiner-core` acceptance gate (significant Pareto-dominance on the behavioral dimension + non-regression on all named dimensions, one-sided z-test at α=0.05). Minting a predicate URI is permanent once the first Statement is signed into the Rekor transparency log; the SAK charter Q3 binding requires this Class-1 ADR before any such mint. This record preserves the verbatim 7-seat positions and the binding conditions, so a future reader can reconstruct why each field, the URL shape, and the production trigger landed where they did.

## 2. Why a council, not a single review

The predicate URI string, its payload field set, and its staging→production flip are each effectively immutable once a row hits production Rekor: walking any of them back means a predicate-version migration plus every downstream verifier reconfiguring. That asymmetry — and the genuine value-tradeoffs (CISO minimal-signed-surface vs CTO/CSO complete-provenance; CMO authority-claim vs CFO immutability-cost) — is what the 7-seat adversarial pattern exists for.

## 3. Synthesis lenses

1. **The arena** — APIs · CLIs · MCP · agents · SKILL.md · 2. **Both sides** — client + server eval · 3. **The transformation pipeline** · 4. **Composable partial attestation** — each predicate row is a valid standalone entry, silence ≠ failure (load-bearing here).
