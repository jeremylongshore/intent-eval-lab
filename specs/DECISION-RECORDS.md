# Decision Records ledger — append-only hash chain

Append-only, hash-chained ledger of every `000-docs/NNN-AT-DECR-*.md` Decision
Record. Each entry commits to its predecessor (Certificate-Transparency-style),
so any edit to an earlier DR — or to any earlier ledger entry — invalidates every
subsequent `entry_hash` and the **HEAD**. This gives the platform's governance
substrate (DR-010 Q3 GC binding: "permanent verbatim 7-seat record") the same
tamper-evidence the platform requires of its own attestations.

- **Generator** — `scripts/build-dr-ledger.sh` (single source of truth for the

- **Verification** — `scripts/build-dr-ledger.sh --verify` recomputes the chain
  from the live DR files and fails if this ledger is stale (a DR was edited in
  place, or the ledger wasn't regenerated after a new DR landed).

## Chain construction

For each AT-DECR doc, in ascending filename order (the `NNN-` prefix is the
canonical append-only sequence):

```
content_hash = sha256( raw bytes of the .md file )
entry_hash   = sha256( "<doc>\n<content_hash>\n<prior_entry_hash>\n" )
```

The genesis `prior_entry_hash` is 64 zeros. The **HEAD** is the `entry_hash` of
the last entry — the single value an external attestation anchors. See
[`docs/dr-ledger-anchoring.md`](../docs/dr-ledger-anchoring.md) for the
cosign-keyless → STAGING-Rekor anchoring procedure.
